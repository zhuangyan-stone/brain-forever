# 触摸翻页组件化方案

## 1. 现状分析

当前触摸翻页逻辑直接内嵌在 [`chat-ui.js`](../../frontend/static/chat-ui.js:323) 的 `showSources()` 函数中（`type === 'web'` 分支），约 350 行代码。主要问题：

| 问题 | 说明 |
|------|------|
| **耦合度高** | 触摸事件处理、三栏布局、圆点导航、翻页动画全部揉在一个函数闭包内 |
| **不可复用** | 如果未来其他面板（如图片轮播、知识库分页）也需要触摸翻页，必须复制粘贴 |
| **状态分散** | `currentPage`、`totalPages`、`isAnimating`、`touchStartX` 等变量散落在闭包中 |
| **DOM 创建与逻辑混合** | slider 的 DOM 创建、事件绑定、数据填充全部在同一个函数中 |

## 2. 设计目标

1. **函数化**：将触摸翻页的核心逻辑抽象为可独立调用的函数
2. **组件化**：提供 `SwipePager` 类，封装 DOM 创建 + 状态管理 + 事件处理
3. **可配置**：通过 options 参数自定义分页大小、动画时长、阈值等
4. **可扩展**：支持自定义渲染函数（`renderPage`），适应不同内容类型
5. **零侵入**：不改变现有 `showSources()` 的调用方式，只重构内部实现

## 3. 架构设计

## 4. 详细设计

### 4.1 SwipePager 类

```javascript
// frontend/static/components/swipe-pager.js

export class SwipePager {
    /**
     * @param {HTMLElement} container - 放置 slider 的容器元素
     * @param {object} options
     * @param {number} options.totalPages - 总页数
     * @param {function(HTMLElement, number): void} options.renderPage - 渲染单页内容的回调
     *        pane: 要填充的 pane 元素（左/中/右栏）
     *        pageIndex: 页码（可能为 null，表示空栏）
     * @param {function(number): void} [options.onPageChange] - 翻页完成后的回调
     * @param {number} [options.pageSize] - 每页条目数（可选，仅用于信息展示）
     * @param {number} [options.swipeThreshold=0.15] - 翻页阈值（占容器宽度的比例）
     * @param {number} [options.animationDuration=250] - 动画时长（毫秒）
     * @param {boolean} [options.showDots=true] - 是否显示底部圆点导航
     * @param {string} [options.dotsClass='swipe-pagination-dots'] - 圆点导航的 CSS 类名
     */
    constructor(container, options) { ... }

    /**
     * 挂载组件：创建 DOM、绑定事件、渲染初始页
     * @param {number} [initialPage=0] - 初始页码
     */
    mount(initialPage = 0) { ... }

    /**
     * 跳转到指定页（带动画）
     * @param {number} page
     */
    slideTo(page) { ... }

    /**
     * 直接渲染指定页（无动画）
     * @param {number} page
     */
    renderPage(page) { ... }

    /**
     * 更新总页数（数据变化时调用）
     * @param {number} totalPages
     */
    updateTotal(totalPages) { ... }

    /**
     * 销毁组件：解绑事件、清理 DOM
     */
    destroy() { ... }

    /** 获取当前页码 */
    get currentPage() { ... }
}
```

### 4.2 内部状态管理

```javascript
// SwipePager 内部状态（不暴露给外部）
this._state = {
    page: 0,           // 当前页码
    total: 0,          // 总页数
    isAnimating: false, // 动画进行中
    isSwiping: false,   // 手指正在滑动
    touchStartX: 0,     // touchstart 时的 X 坐标
};
```

### 4.3 DOM 结构

由 `SwipePager` 自动创建：

```html
<div class="swipe-container">          <!-- container（由外部传入） -->
    <div class="swipe-slider">          <!-- 三栏 slider -->
        <div class="swipe-pane"></div>  <!-- 左栏（上一页） -->
        <div class="swipe-pane"></div>  <!-- 中栏（当前页） -->
        <div class="swipe-pane"></div>  <!-- 右栏（下一页） -->
    </div>
    <div class="swipe-pagination-dots">  <!-- 圆点导航 -->
        <span class="swipe-dot active"></span>
        <span class="swipe-dot"></span>
        ...
    </div>
</div>
```

### 4.4 事件处理

所有事件绑定在 `mount()` 中完成，`destroy()` 中解绑：

| 事件 | 目标 | 处理逻辑 |
|------|------|----------|
| `touchstart` | `.swipe-container` | 记录 `touchStartX`，重置 `isSwiping` |
| `touchmove` | `.swipe-container` | 计算偏移，跟随手指移动 slider |
| `touchend` | `.swipe-container` | 判断是否超过阈值，翻页或回弹 |
| `click`（委托） | `.swipe-pagination-dots` | 点击圆点跳转到对应页 |
| `resize` | `window` | 重新计算容器宽度（可选） |

### 4.5 三栏布局核心算法

复用现有方案，封装为内部方法：

```javascript
// 基准位置：translateX(-33.33%) → 显示中栏（当前页）
// 左滑（下一页）：从 -33.33% 向 -66.66% 移动
// 右滑（上一页）：从 -33.33% 向 0 移动

_getBaseOffset() { return -33.33; }
_getNextOffset()  { return -66.66; }
_getPrevOffset()  { return 0; }

// 手指跟随偏移计算
// translateX(p%) 移动距离 = p/100 * slider宽度 = p/100 * 容器宽度 * 3
// 要跟随手指移动 dx 像素：dx = p/100 * containerWidth * 3 → p = dx/containerWidth * 33.33
_calcFollowOffset(dx, containerWidth) {
    return dx / containerWidth * 33.33;
}
```

### 4.6 预渲染策略

```javascript
_fillTriple(page) {
    // 左栏 = page - 1（或 null）
    this._options.renderPage(this._panes.left, page > 0 ? page - 1 : null);
    // 中栏 = page
    this._options.renderPage(this._panes.center, page);
    // 右栏 = page + 1（或 null）
    this._options.renderPage(this._panes.right, page < this._state.total - 1 ? page + 1 : null);
}
```

## 5. 使用示例

### 5.1 在 showSources 中使用（重构后）

```javascript
// chat-ui.js 中 showSources() 的 web 分支
import { SwipePager } from './components/swipe-pager.js';

// ... 创建 itemsContainer 和 body ...

const pager = new SwipePager(itemsContainer, {
    totalPages: Math.ceil(sources.length / PAGE_SIZE),
    renderPage: (pane, pageIndex) => {
        pane.innerHTML = '';
        if (pageIndex === null || pageIndex < 0) return;
        const start = pageIndex * PAGE_SIZE;
        const end = Math.min(start + PAGE_SIZE, sources.length);
        for (let i = start; i < end; i++) {
            pane.appendChild(createSourceItem(sources[i]));
        }
    },
    showDots: true,
    dotsClass: 'sources-pagination-dots',
});
pager.mount(0);
```

### 5.2 未来其他场景复用

```javascript
// 图片轮播
const imagePager = new SwipePager(imageContainer, {
    totalPages: images.length,
    renderPage: (pane, idx) => {

        img.src = images[idx].url;
        pane.appendChild(img);
    },
    swipeThreshold: 0.2,
});
imagePager.mount(0);

// 知识库分页

    },

ragPager.mount(0);
```

## 8. 设计决策记录

| 决策 | 选项 | 选择理由 |
|------|------|----------|
| 类 vs 函数 | 类 | 需要管理状态（page、isAnimating 等），类更自然 |
| 三栏 vs 两栏 | 三栏（保持现有） | 已验证的成熟方案，左右滑对称，touchmove 无需动态填充 |
| renderPage 回调 vs 继承 | 回调 | 组合优于继承，更灵活，使用者无需创建子类 |
| 圆点导航内置 vs 外部 | 内置（可关闭） | 90% 场景都需要，减少使用者工作量 |
| passive 事件 | 是 | 不调用 preventDefault，提升触摸滚动性能 |

## 9. 未来扩展

- **鼠标拖拽支持**：增加 `mousedown`/`mousemove`/`mouseup` 事件，桌面端也可拖拽翻页
- **键盘支持**：左右方向键翻页
- **无限循环**：`loop: true` 选项，首尾相连
- **自动播放**：`autoplay: true` 选项，定时自动翻页
- **指示器样式**：支持数字指示器、进度条等替代圆点

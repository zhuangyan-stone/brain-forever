# 触摸滑动翻页实现方案

## 概述

在 Web 搜索结果的 sources-panel 中，通过触摸滑动实现分页导航。用户可以在智能手机触屏上左右滑动来切换页面，同时支持点击底部圆点导航。

## 核心设计：三栏布局

### 结构

采用三栏并排的 slider 布局，替代传统的两栏方案：

```
┌─────────────┬─────────────┬─────────────┐
│  左栏       │  中栏       │  右栏       │
│  上一页     │  当前页     │  下一页     │
│  (prev)     │  (current)  │  (next)     │
└─────────────┴─────────────┴─────────────┘
←───────────── 300% ─────────────→
```

- slider 宽度 = 容器宽度 × 300%
- 每栏宽度 = 容器宽度（33.33%）
- 通过 `overflow: hidden` 容器裁剪，一次只显示一栏

### 定位基准

使用 CSS `translateX` 控制显示哪一栏：

| translateX 值 | 显示内容 | 说明 |
|---|---|---|
| `translateX(0)` | 左栏（上一页） | 右滑翻页目标 |
| `translateX(-33.33%)` | 中栏（当前页） | **基准位置** |
| `translateX(-66.66%)` | 右栏（下一页） | 左滑翻页目标 |

> `translateX` 百分比是相对于元素自身宽度。slider 宽度 = 容器宽度 × 3，所以 `translateX(-33.33%)` 移动距离 = 容器宽度 × 3 × 33.33% = 容器宽度，正好显示中栏。

### 预渲染策略

翻页完成后，立即预渲染三栏内容：

```
左栏 = currentPage - 1  (或 null，首页时)
中栏 = currentPage       (当前页)
右栏 = currentPage + 1  (或 null，末页时)
```

这样无论用户左滑还是右滑，目标页内容已经就绪，touchmove 中无需动态填充 DOM，避免闪烁和性能问题。

## 触摸事件处理

### touchstart

```javascript
touchStartX = e.changedTouches[0].screenX;
isSwiping = false;
```

记录起始触摸位置，重置滑动状态。

### touchmove — 手指跟随

计算手指移动距离 `dx = currentX - touchStartX`：

```
translateX 偏移 = -33.33 + dx / containerWidth * 33.33
```

**推导：**
- `translateX(p%)` 移动距离 = `p/100 × slider宽度` = `p/100 × 容器宽度 × 3`
- 要跟随手指移动 `dx` 像素：`dx = p/100 × containerWidth × 3`
- 所以：`p = dx / containerWidth × 33.33`

**左滑（dx < 0）：**
- 从 `-33.33%` 向 `-66.66%` 移动
- 右栏（下一页）从右侧滑入视野
- 偏移范围：`max(-33.33 + dx/W*33.33, -60)` — 留余量给 touchend 的 `-66.66%`

**右滑（dx > 0）：**
- 从 `-33.33%` 向 `0` 移动
- 左栏（上一页）从左侧滑入视野
- 偏移范围：`min(-33.33 + dx/W*33.33, -5)` — 留余量给 touchend 的 `0`

**边界处理：**
- 首页（currentPage === 0）禁止右滑
- 末页（currentPage === last）禁止左滑

### touchend — 翻页/回弹

**翻页判定：** 手指移动距离超过容器宽度的 15%

| 方向 | 条件 | 目标 translateX | 显示内容 |
|---|---|---|---|
| 左滑翻页 | dx < -threshold | `-66.66%` | 右栏（下一页） |
| 右滑翻页 | dx > threshold | `0` | 左栏（上一页） |
| 回弹 | 未超过阈值 | `-33.33%` | 中栏（当前页） |

**翻页完成（setTimeout 280ms）：**
1. 更新 `currentPage`
2. 调用 `fillTriple()` 重新预渲染三栏
3. 重置 slider 到基准位置 `translateX(-33.33%)`
4. 更新圆点导航

使用 `setTimeout` 而非 `transitionend` 事件，因为当 touchmove 已将 translateX 设为目标值时，transition 不会触发，导致 `transitionend` 不执行。

## 点击圆点导航

`slideToPage(page)` 函数处理点击圆点的动画翻页：

**向后翻（下一页）：**
```
translateX(0) → translateX(-33.33%)
```
先显示左栏（=当前页），再动画到中栏（=目标页），中栏从右侧滑入。

**向前翻（上一页）：**
```
translateX(-66.66%) → translateX(-33.33%)
```
先显示右栏（=当前页），再动画到中栏（=目标页），中栏从左侧滑入。

## CSS 样式

```css
.sources-items-container {
    overflow: hidden;
    touch-action: pan-y;       /* 允许垂直滚动，捕获水平滑动手势 */
    user-select: none;         /* 防止滑动时选中文本 */
}

.sources-slider {
    display: flex;
    width: 300%;
    will-change: transform;    /* GPU 加速 */
}

.sources-slider-pane {
    width: 33.33%;
    flex-shrink: 0;
    box-sizing: border-box;
}
```

## 关键变量

| 变量 | 类型 | 说明 |
|---|---|---|
| `currentPage` | number | 当前页码（0-based） |
| `totalPages` | number | 总页数 |
| `PAGE_SIZE` | number | 每页条目数（固定为 5） |
| `isAnimating` | boolean | 动画进行中，阻止新的触摸事件 |
| `isSwiping` | boolean | 手指正在滑动中 |
| `touchStartX` | number | touchstart 时的屏幕 X 坐标 |

## 演进历史

1. **两栏布局 + 简单触摸**：左滑成功，右滑 `translateX(50%)` 导致内容移出视野
2. **两栏布局 + fillPaneRight**：右滑 touchmove 中填充右半，但 `translateX(50%)` 根本性错误
3. **两栏布局 + 镜像对称**：右滑时交换左右内容，从 `-50%` 向 `0` 移动，但缺少强制回流导致卡住
4. **三栏布局（最终方案）**：预渲染前后两页，左右滑完全对称，touchmove 无需动态填充 DOM

# personal-traits

| 字段名 | 含义 | 说明 | AI 生成 |
|-|-|-|-|
| ID | ID | 记录唯一值 | | 
| topic | 话题 | 当前用户讨论的话题是什么 |✓|
| inference_method | 推导模式 | 显性表达（explicit-traits）、隐性表达 （implicit-traits） | ✓ |
| nature | 特征性质 | 陈述客观特征（objective-traits）或表达主观感受（subjectivity-traits） | ✓ |
| conclution | 特征主题 | 一句简短的话描述一个特征 | ✓ |
| scenario | 应用场景 | 捕获特征时的对话场景（上下文） | ✓ |
| domain | 领域 | 特征本身的内容领域 | ✓ |
| category | 顶层分类 | 9 个分类 | ✓ |
| source | 来源 | 对用于产生用户特征的原文引用（明确的句子）或相关归纳（跨段落的文字归纳、缩写） | ✓ |
| confidence | 置信度 | 0.1 ~ 1.0 | ✓ |
| half_life | 半衰期 | 特征数据的时效性和变化频率。short/medium/long | ✓ |
| create_at | 生成时间 | |
| update_at | 更新时间 | |



本项目中，将有一个专用的 “旁观者/分析者/归纳者” LLM客户端，称为 “traits-agent”。它不和用户对话，但是会观察用户与AI（另一个 agent ，称为 chat-agent）之间的对话内容，生成一或多条可信的用户特征。特征需要包括上表中的  “inference_method、nature、topic、scenario 、domian、cate”

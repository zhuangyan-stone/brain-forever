# 数据库连接架构设计

> 最后更新: 2026-07-05

## 一、核心设计原则

### 1.1 `users.db` — 全局单例，提前打开，延迟关闭

**`users.db`** 是全局共享的用户/角色数据库，采用 **程序启动时提前打开，程序退出时最后关闭** 的策略。

- 在 HTTP 服务器监听前打开，确保所有请求到来时数据库就绪
- 在 HTTP 服务器停止监听后关闭，避免程序退出时还有未完成的请求
- 通过全局单例 [`theUserStore`](internal/store/the_user_store.go) 访问

### 1.2 `*.chats.db` / `*.brain.db` — 按需打开，用完即关

**对话**和**特征向量**数据库不在任何对象生命周期内常驻，而是在每次需要时临时打开，操作完成后立即关闭。

- 通过 [`dbcfg`](internal/store/dbcfg/dbcfg.go) 子包统一管理
- 包级全局函数，无需依赖注入
- 唯一入参：`userSN`（User Serial Number）

### 1.3 Session 的 userSN 语义

- **`userSN` 非空** = 已登录
- **`userSN` 为空** = 未登录（API 应拒绝服务）
- session 不持有任何数据库连接（无 xxxStorage 字段）
- **不再存在匿名用户**，所有用户都有 `userSN`

## 二、数据库文件清单

| 文件模式 | 用途 | 生命周期 | 打开时机 |
|---------|------|---------|---------|
| `localdb/users.db` | 全局用户注册 + 角色 | **全局单例** | **程序启动时**（HTTP 监听前） |
| `localdb/{userSN}.chats.db` | 用户对话 | 按需 | 每次 API 请求 |
| `localdb/{userSN}.brain.db` | 用户特征向量 | 按需 | 特征提取/画像时 |

### 命名规则

- 所有用户直接使用 `userSN` 作为文件名
- 如 `test_user_001.chats.db`、`anonymous.chats.db`
- 不再有 `anon_*` 前缀的文件

### 3.2 类型描述

#### [`ChatStore`](internal/store/chats.go:14)
- 每个用户拥有独立的 SQLite 文件
- 存储：对话 session、消息、网页来源、标签、收藏
- 方法：`InsertChat`、`InsertMessage`、`ListChats`、`Close` 等

#### [`BrainStore`](internal/store/traits.go:54)（原 `VectorStore`）
- 每个用户拥有独立的 SQLite 文件
- 存储：个人特征条目、向量索引、关键词
- 方法：`AddTrait`、`SearchByVector`、`SearchByKeyword`、`Close` 等

#### [`UserStore`](internal/store/users.go:37)
- 全局共享 `users.db`，所有用户共用
- 存储：用户信息、角色
- 方法：`CreateUser`、`VerifyPassword`、`Close` 等

#### [`theUserStore`](internal/store/the_user_store.go)（全局单例）
- 封装 `UserStore`，提供全局访问点
- `InitTheUserStore(dbDir)` — 程序启动时调用
- `TheUserStore()` — 任意位置获取
- `CloseTheUserStore()` — 程序退出时调用

#### [`dbcfg`](internal/store/dbcfg/dbcfg.go)（子包，全局状态）
- 替代旧 `DBOpener`，包级函数模式（无结构体）
- `InitLocalChatDB` / `InitLocalBrainDB` — 打开 + 初始化表结构（用于首次登录）
- `OpenLocalChatDB` / `OpenLocalBrainDB` — 仅打开，不检查表结构（热路径性能优化）
- `CloseLocalChatDB` / `CloseLocalBrainDB` — 用完关闭
- 持有 `theDBDir`、`theEmbedderDim`、`theLogger`
- `IsRealUser` 已移除，前端直接传 `user_sn`

## 四、依赖注入简化对比

### 旧模式（依赖注入传递 dbOpener）

```
main → InitAgent(dbOpener) → NewChatHandler(dbOpener) → NewSessionManager(dbOpener)
                                                              ↓
                                                         SessionManager.dbOpener
                                                              ↓
                                                         switchToUser 需要传 localDB
                                                         DeleteMessage 通过 sm.dbOpener
```

### 新模式（包级全局 dbcfg）

```
main → dbcfg.InitDBConfig(...)    ← 一次初始化，全局生效
     → agent.InitAgent()          ← 不再需要 db 参数
         → NewChatHandler()       ← 不再需要 db 参数
             → sessionManager     ← 不再需要 db 参数
                 → handler 中直接调用：
                     dbcfg.OpenLocalChatDB(userSN)
                     dbcfg.CloseLocalChatDB(chatStore)
```

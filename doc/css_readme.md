# 第二大脑 · 主题 CSS 变量参考

> 本文档列出主题系统中所有 CSS 自定义属性（Custom Properties）及其用途，
> 供主题设计者参考。每个外源主题的 `theme.css` 通过覆盖这些变量来改变外观。

---

## 一、全局布局常量 — `:root`

定义于 [`frontend/static/theme.css`](../static/theme.css) 的 `:root` 中，**不随主题变化**。
外源主题**无需**重新定义它们。

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `--radius` | `12px` | 全局圆角大小，控制卡片、按钮、对话框等元素的圆角 |
| `--max-width` | `960px` | 内容区最大宽度，用于 `#app` 容器 |
| `--pure-black` | `#000000` | 绝对纯黑，用于需要精确黑色的场景 |
| `--pure-white` | `#FFFFFF` | 绝对纯白，用于需要精确白色的场景 |

---

## 二、主题色彩变量 — `[data-theme]`

定义于 `[data-theme="light"]` 或 `[data-theme="dark"]` 选择器内。
**外源主题通过覆盖这些变量来替换内置主题的配色。**

### 2.1 背景色系

| 变量 | 说明 |
|------|------|
| `--bg-primary` | **主背景色** — 页面最底层的背景 |
| `--bg-image` | **背景图片** — `url('...')` 或 `none`，覆盖在主背景之上 |
| `--bg-secondary` | **次要背景色** — 面板、侧栏、分割区域的背景 |
| `--bg-user` | **用户消息气泡背景** — 用户发送消息的聊天气泡底色 |
| `--bg-assistant` | **AI 消息气泡背景** — AI 助手回复消息的气泡底色 |
| `--bg-input` | **输入框背景** — 底部输入区域的背景色 |
| `--switch-track-bg` | **开关轨道背景** — 切换开关/滑块的滑轨底色 |
| `--bg-code` | **代码块背景** — Markdown 代码块 `<pre><code>` 的背景色 |

### 2.2 文字色系

| 变量 | 说明 |
|------|------|
| `--text-primary` | **主要文字色** — 正文、标题等核心文字 |
| `--text-secondary` | **次要文字色** — 辅助信息、元数据文字 |
| `--text-muted` | **弱化文字色** — 最次要的提示性文字，如时间戳 |

### 2.3 强调/交互色

| 变量 | 说明 |
|------|------|
| `--accent` | **强调色** — 品牌色、链接色、可交互元素的默认状态 |
| `--accent-hover` | **强调色悬停态** — 鼠标悬停/聚焦时的强调色变体 |

### 2.4 边框与阴影

| 变量 | 说明 |
|------|------|
| `--border` | **边框色** — 容器、卡片、分割线的边框颜色 |
| `--shadow` | **外阴影色** — 元素投影的 `rgba()` 颜色值（不含偏移/模糊） |
| `--shadow-inset` | **内阴影色** — 元素内阴影的完整 `inset` 值 |

### 2.5 语义状态色

| 变量 | 说明 |
|------|------|
| `--success-text` | **成功文字色** — 操作成功、完成状态的文字颜色 |
| `--error-bg` | **错误背景色** — 错误提示的背景色（通常用半透明） |
| `--error-border` | **错误边框色** — 错误提示的边框色（通常用半透明） |
| `--error-text` | **错误文字色** — 错误提示的文字颜色 |
| `--warning-bg` | **警告背景色** — 警告提示的背景色 |
| `--warning-border` | **警告边框色** — 警告提示的边框色 |
| `--warning-text` | **警告文字色** — 警告提示的文字颜色 |

### 2.6 组件色

| 变量 | 说明 |
|------|------|
| `--tooltip-bg` | **Tooltip 背景色** — 工具提示气泡的背景 |
| `--tooltip-text` | **Tooltip 文字色** — 工具提示气泡的文字 |
| `--logo-circle-bg` | **Logo 圆形背景** — 登录页 logo 外围圆形底色的背景（亮色透明、暗色白色） |
| `--btn-highlight-bg` | **高亮按钮背景** — 特殊高亮操作按钮的底色 |
| `--btn-highlight-color` | **高亮按钮文字/图标色** — 高亮按钮上的文字或图标颜色 |
| `--btn-highlight-hover-bg` | **高亮按钮悬停背景** — 高亮按钮悬停时的背景色 |
| `--btn-danger-bg` | **危险按钮背景** — 删除/危险操作的按钮底色 |
| `--btn-danger-color` | **危险按钮文字色** — 危险按钮上的文字颜色 |
| `--btn-danger-hover-bg` | **危险按钮悬停背景** — 危险按钮悬停时的背景色 |

## 三、如何创建一个新主题

1. 在 [`frontend/themes/`](./) 下新建文件夹，命名规则：`<英文id>-<light|dark>`（如 `mytheme-light`）。
2. 在该文件夹下创建 `theme.css`，用目标 `[data-theme]` 选择器包裹变量定义：

```css
[data-theme="light"] {
    --bg-primary:       #...;
    --bg-secondary:     #...;
    /* ...仅定义需要覆盖的色彩变量，布局常量由 :root 自动提供... */
}
```

3. （可选）添加 `bg.svg` 作为背景图片。
4. 在 [`manifest.json`](./manifest.json) 的 `themes` 数组中注册新主题。
5. 用户通过主题对话框选择后自动生效。

> **注意**：外源主题只需定义**色彩变量**，`--radius`、`--max-width`、`--pure-black`、`--pure-white` 等布局常量会自动从内置 `:root` 继承。

---

## 四、CSS 加载顺序

```
:root 布局常量（base.css / theme.css :root）
  └─ [data-theme] 内置主题色彩（theme.css）
       └─ [data-theme] 外源主题色彩（themes/*/theme.css）— 后加载，覆盖内置
```

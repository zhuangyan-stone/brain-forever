SSE 流式渲染：原生 JS vs Alpine.js 代码对比

点击"模拟 SSE 流"按钮，两边同时接收相同数据，观察实现方式差异。

① 原生 JS 实现（当前项目的做法）

渲染策略：throttleRender(180ms) — 节流 → innerHTML 全量替换 → autoScroll

② Alpine.js 实现

渲染策略：x-text + setTimeout 节流 — 仍需手动 throttle + autoScroll

关键代码对比

当前项目（原生 JS）
// chat-sse-responser.js onText(event) { // 1. 累积到 buffer this.session.streamingMsg.content += event.content; // 2. 节流渲染 + autoScroll throttleRender( this.session, // timer holder this.session.contentDiv,// target DOM () => this.session .streamingMsg.content // getText ); } // chat-ui.js — throttleRender export function throttleRender( timerHolder, targetEl, getText ) { if (timerHolder.renderTimer) return; timerHolder.renderTimer = setTimeout(()=>{ timerHolder.renderTimer = null; targetEl.innerHTML = renderMarkdown(getText()); autoScrollToBottom(); }, state.RENDER_INTERVAL); // 180ms } // —— 关键设计 —— // • timer 附着在 session 对象上 // 切换对话不会丢失计时 // • renderMarkdown 处理 Markdown // • autoScrollToBottom 处理滚动 // 三个职责都在原生 JS 中清晰控制
Alpine.js 实现
<!-- HTML 模板 --> <div x-data="{ content: '', // 响应式数据 _buffer: '', // 累积缓冲区 _timer: null, onChunk(chunk) { this._buffer += chunk; if (this._timer) return; this._timer = setTimeout(() => { // Alpine 响应式赋值 this.content = this._buffer; this._timer = null; // ⚠️ autoScroll 仍需手动 this.$nextTick(() => { const box = this.$el .querySelector('.chat-box'); box.scrollTop = box.scrollHeight; }); }, 180); } }"> <div x-text="content"></div> </div> // —— 与原生 JS 的差异 —— // • 累积 + 节流 + 渲染 三个动作 // 仍然一个都不能少 // • 多了 Alpine 的响应式开销 // • x-text 不支持 Markdown 渲染 // 仍需手动 innerHTML // • autoScroll 无变化，代码量相同

为什么 Alpine.js 没有本质改善

环节 当前做法 Alpine 能做吗
数据累积
SSE chunk → buffer		 streamingMsg.content += chunk
手动，清晰，无框架依赖		 ❌ 不能简化。你仍然需要 this._buffer += chunk。Alpine 的响应式只要一赋值就会触发重渲染，不能在每次 chunk 到来时都 set content。
渲染节流
180ms 一次，避免高频刷新		 setTimeout + renderTimer
准确控制渲染时机		 ❌ 不能简化。你仍然需要 setTimeout 做节流。Alpine 的 x-effect 是同步的，无法替代节流逻辑。
Markdown 渲染
content → HTML		 innerHTML = renderMarkdown(text)
直接调用 remarkable.js		 ❌ 不能简化。x-text 只支持纯文本；x-html 没有内置的 Markdown 处理。你仍然需要手动调用 renderMarkdown()。
自动滚动
内容变化后 scrollToBottom		 scrollContainer.scrollTop = height ❌ 不能简化。Alpine 没有"DOM 内容变化后自动滚动"的功能。你仍然需要 this.$nextTick(() => box.scrollTop = box.scrollHeight)。
多会话隔离
timer/数据各会话独立		 ChatSession 类实例持有自己的 timer 和 streamingMsg ⚠️ 做到但不更简单。需要用多个 Alpine 组件实例（x-data）或 store 来隔离会话状态，但 timer 管理逻辑量相同。

# 出错信息格式规范

## 总则

1. 错误消息（`fmt.Errorf`、`logger.Errorf`、`t.Fatalf`、`log.Printf`、`fmt.Sprintf`、`panic(fmt.Sprintf(...))` 等）中的 `: %v` 和 `: %w` **不得使用冒号**作为前缀，应使用句号。
2. 中文使用中文句号 `。`，英文使用英文句点 `. `（带尾随空格）。
3. 若错误消息后**没有后续内容**，则尾部**不添加任何标点符号**（除非确实需要表达强烈置疑时带问号，或表达强烈感情时加 `!!!`）。
4. 若有后续内容，则在说完完整一句话后接句号，再接后面的信息。

## 规则详解

### 规则 1：Go 代码中的 `: %v` / `: %w`

**错误：**
```go
fmt.Errorf("failed to create user: %w", err)
logger.Errorf("BEGIN transaction failed: %v", err)
```

**正确：**
```go
fmt.Errorf("failed to create user. %w", err)
logger.Errorf("BEGIN transaction failed. %v", err)
```

### 规则 2：SQL 日志的 `]: %v`

SQL 日志中，错误详情应**换行显示**，而不是紧跟在 `]` 之后。

**错误：**
```go
s.logger.Errorf("SQL [%s] args=[id=%d]: %v", sqlStr, id, err)
// 输出: SQL [SELECT ...] args=[id=1]: ERROR: value too long (SQLSTATE 22001)
```

**正确：**
```go
s.logger.Errorf("SQL [%s] args=[id=%d]:\n%v", sqlStr, id, err)
// 输出:
// SQL [SELECT ...] args=[id=1]
// ERROR: value too long (SQLSTATE 22001)
```

### 规则 3：中文 i18n 模板中的冒号

中文语言包（`lang/zh-CN*.toml`）中的错误消息，模板变量前的全角冒号 `：` 应替换为中文句号 `。`（后面无空格）。

**错误：**
```toml
other = "读取个人特征失败：{{.Error}}"
other = "缺少参数：{{.Param}}"
```

**正确：**
```toml
other = "读取个人特征失败。{{.Error}}"
other = "缺少参数。{{.Param}}"
```

### 规则 4：英文 i18n 模板中的冒号

英文语言包（`lang/en*.toml`）中的错误消息，模板变量前的半角冒号 `: ` 应替换为英文句点加空格 `. `。

**错误：**
```toml
other = "Failed to read traits: {{.Error}}"
other = "validation failed: {{.Error}}"
```

**正确：**
```toml
other = "Failed to read traits. {{.Error}}"
other = "validation failed. {{.Error}}"
```

### 规则 5：用户可见消息的尾部句号

直接展示给最终用户的短消息（工具描述、状态提示等），句末**不加句号**。

**错误：**
```toml
[all_messages_loaded]
other = "所有对话消息已加载完毕，没有更多内容。"
```

**正确：**
```toml
[all_messages_loaded]
other = "所有对话消息已加载完毕，没有更多内容"
```

### 规则 6：UI 标签的尾部冒号

UI 标签（如 `用户：`、`AI：`、`User: `、`AI: `）中去掉尾部冒号，因为消息内容紧随其后，标签本身无需标点结尾。

**错误：**
```toml
[label_user]
other = "用户："
```

**正确：**
```toml
[label_user]
other = "用户"
```

### 规则 7：LLM 提示词（System Prompt）中的冒号

LLM 系统提示词（`system_prompt.toml`）中的冒号是自然语言的一部分，用于引导说明或引入变量值（如 `"当前标题是：{{.Title}}"`），**不受本规范约束**，保持原有自然语言风格不变。

## 判断标准速查表

| 上下文 | 冒号用法 | 示例 |
|--------|----------|------|
| Go `fmt.Errorf` / `logger.Errorf` 中的 `%v`/`%w` | 改为句号 | `"failed. %w"` |
| SQL 日志中的 `%v` | 改为换行 | `"...]:\n%v"` |
| 中文 i18n `{{.Error}}`/`{{.Param}}` 前 | 中文句号 `。` | `"失败。{{.Error}}"` |
| 英文 i18n `{{.Error}}` 前 | 句点+空格 `. ` | `"failed. {{.Error}}"` |
| 用户可见短消息尾部 | 不加标点 | `"已加载完毕"` |
| UI 标签尾部 | 不加标点 | `"用户"` |
| LLM 提示词 | 保留自然语言 | `"当前标题是：{{.Title}}"` |

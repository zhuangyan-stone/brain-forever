# 中文资源文件标点规范

## 适用范围

本规范适用于项目中所有中文（`lang/zh-CN/`）资源文件（`.toml`），以及前端（`frontend/`）HTML/JS 中直接写死的 UI 中文文本。

## 标点替换规则

### 1. 逗号 `,` → `，`

- 中文文本中，英文逗号必须改为中文全角逗号。
- 原英文逗号后如有空格，一并删除。

```diff
- "核心特质关键词列表，如 ['理性', '真诚', '独立']"
+ "核心特质关键词列表，如 ['理性'，'真诚'，'独立']"
```

### 2. 冒号 `:` → `：`

- 中文文本中，英文冒号必须改为中文全角冒号。

```diff
- "type: A=时间, B=地点"
+ "type：A=时间，B=地点"
```

### 3. 圆括号 `()` → `（）`

- 中文文本中的圆括号必须使用中文全角括号。
- 例外：**数学表达式中的优先级括号**（如 `(a + b) * c`）保留英文括号。

```diff
- "正常身体测量值归(1)"
+ "正常身体测量值归（1）"

- "物件(包括动物、植物等)"
+ "物件（包括动物、植物等）"
```

### 4. 省略号 `...` / `......` / `…` → `……`

- 中文文本中，省略号必须使用两个连续的中文省略号 `……`（U+2026 U+2026），而非英文句点 `...` 或 `......`，也非单个省略号 `…`。

```diff
- "他在思考...嗯...也许吧"
+ "他在思考……嗯……也许吧"
```

```diff
- "也许...大概...是这样"
+ "也许……大概……是这样"
```

```diff
- "你确定…？"
+ "你确定……？"
```

### 5. 保留不变的情况

以下符号在中文资源文件中保留英文原样：

| 符号 | 说明 | 示例 |
|------|------|------|
| `=` | 表示相等/赋值 | `A=时间` |
| `~` | 表示范围 | `5~10个` |
| `-` | 表示范围/连字符 | `A-F`、`天~周` |
| `/` | 表示或/分隔 | `偏好/癖好`、`好喝/好吃` |
| `[]` | 数组/代码语法 | `['理性', '真诚']` |
| `"` | 代码中的引号 | `\"Hello world\"` |
| `_` | 代码中的下划线 | `category_id` |
| 代码示例 | 代码片段内的标点 | 保持原样 |
| URL/路径 | 链接或文件路径 | 保持原样 |

## 检查方法

在项目根目录运行以下命令检查中文资源文件中的英文标点问题：

```bash
# 检查中文后的英文逗号
grep -Pn '[\x{4e00}-\x{9fff}],' lang/zh-CN/**/*.toml

# 检查中文后的英文圆括号
grep -Pn '[\x{4e00}-\x{9fff}]\([^\x{4e00}-\x{9fff}]*[）\)]' lang/zh-CN/**/*.toml

# 检查中文后的英文冒号
grep -Pn '[\x{4e00}-\x{9fff}]:' lang/zh-CN/**/*.toml
```

## 例外说明

上述规则仅针对**中文文本内容**（即 `other = "..."` 中的自然语言描述）。以下情况不受此规则约束：

- JavaScript 语法中的 `...`（扩展运算符、rest 参数等）
- 模板语法 `{{.Variable}}`
- 代码示例中的标点（如 JSON 格式 `{"key": "value"}`）
- 英文术语/品牌名中的标点（如 `C++`、`iPhone15`）

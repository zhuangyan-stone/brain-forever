# Go 源代码注释语言规范

## 规则

Go 源代码文件（`.go`）中的注释必须使用 **英文**。

### 适用范围

Go 源代码文件（`.go`）中**所有**注释、说明性文字必须使用英文，无一例外。

包括但不限于：
- 行注释 `// ...`
- 块注释 `/* ... */`
- 函数/方法/类型/常量/变量的 doc comment
- 代码中的任何自然语言说明性文字

### 例外

- `//go:` 指令行
- `//nolint:` 等 linter 指令

### 原因

保持 Go 代码库的注释语言统一为英文，有利于：
- 跨团队协作（非中文母语的开发者也能理解）
- 代码搜索工具（grep、IDE 查找）的统一性
- Go 社区惯例

### 检查方法

```bash
grep -rn '//.*[\x{4e00}-\x{9fff}]' internal/ --include='*.go'
```
